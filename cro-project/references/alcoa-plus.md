# ALCOA+ 原则详解

**全称：** Attributable, Legible, Contemporaneous, Original, Accurate + Complete, Consistent, Enduring, Available  
**应用领域：** GxP（药品生产质量管理规范/药品临床试验质量管理规范）领域的数据完整性管理

---

## 1 ALCOA 原则

ALCOA 原则起源于 FDA 1970 年代的数据完整性指南，是评估数据质量的基础框架。

### 1.1 A — Attributable（可归因性）

**定义：** 每一条数据记录**必须**能够追溯到产生该数据的具体个人。

**核心要求：**
- 电子记录**必须**包含创建者或修改者的身份标识
- 纸质记录**必须**有原始签名或盖章
- **禁止**使用无法识别来源的匿名记录

**实施要点：**
- 电子系统：采用唯一用户账号 + 电子签名
- 纸质记录：原始签名并注明日期
- 采集设备：自动采集时记录设备编号和操作者

### 1.2 L — Legible（清晰、可读）

**定义：** 数据记录**必须**清晰、可读，便于审查和追溯。

**核心要求：**
- 纸质记录**必须**使用永久性墨水，字迹清晰可辨
- 电子数据**必须**以 Human-readable 格式呈现
- 数据**应当**按时间顺序排列

**实施要点：**
- **禁止** 使用难以辨认的缩写或代码（如使用，必须有明确说明）
- 纸质记录出现划改时，**必须**保持原记录可读
- 电子记录导出时，**必须**保证可读性

### 1.3 C — Contemporaneous（同时性）

**定义：** 数据**应当**在其产生的同时被记录。

**核心要求：**
- 观察与记录**应当**同步进行，避免记忆偏差
- 记录时间戳**必须**反映实际发生时间
- **禁止** 事后补填或倒填日期

**实施要点：**
- 如果无法实时记录，**应当**在备注中说明延迟原因
- EDC 系统**应当**自动记录数据录入时间
- 纸质记录仪**应当**校准时间同步

### 1.4 O — Original（原始性）

**定义：** 数据记录**必须**保留原始内容，或有经核证的原始副本。

**核心要求：**
- **必须**保存原始数据（Primary Record）
- 复印件**必须**与原件一致，并注明复印时间和复核人
- 电子数据**必须**保留原始格式和元数据

**实施要点：**
- 源数据核查（SDV）时，**必须**比对原始记录
- 数据迁移时，**必须**保留原始数据或提供迁移验证
- **禁止** 直接在打印件上修改后扫描替代原始记录

### 1.5 A — Accurate（准确性）

**定义：** 数据记录**必须**真实反映实际操作或观察结果。

**核心要求：**
- 数据**必须**经过验证，确保测量和记录的准确性
- 数据录入**必须**双人核对或系统自动校验
- 错误**必须**按规范修正，保留修改轨迹

**实施要点：**
- 仪器设备**必须**定期校准
- 数据录入规则**应当**设置逻辑校验和范围检查
- 异常值**应当**有合理说明

## 2 ALCOA+ 扩展原则

在原有 ALCOA 基础上，增加了四个要素，进一步强化数据完整性保障。

### 2.1 C — Complete（完整性）

**定义：** 所有数据，包括所有测试结果、研究过程记录，**必须**完整保存，不得选择性记录或删除。

**核心要求：**
- **禁止** 隐瞒或删除不利数据（Nature 6: 阴性结果也要发表）
- 完整的数据集包括重复实验、失败实验的记录
- 审计追踪**必须**记录所有操作，包括删除操作

### 2.2 C — Consistent（一致性）

**定义：** 数据记录**必须**内部一致，不同来源的数据**应当**相互印证。

**核心要求：**
- 日期、时间顺序**必须**合理，无逻辑矛盾
- 不同表单间的关联数据**必须**保持一致
- 电子记录与纸质记录**应当**保持一致

**实施要点：**
- **应当**设置跨表单逻辑校验
- 数据迁移后**必须**进行一致性核查
- 发现不一致**必须**及时调查原因

### 2.3 E — Enduring（持久性）

**定义：** 数据记录**必须**以持久的方式保存，确保在整个记录保留期内可读取。

**核心要求：**
- 纸质记录**应当**使用耐久的材料（好质量纸张、永久性墨水）
- 电子数据**必须**采用开放格式或长期可获取的格式
- 数据保留期限**必须**符合法规要求

**实施要点：**
- **应当**避免使用专有格式存储关键数据
- 定期检查数据可读取性
- 数据迁移时**必须**保留元数据和上下文

### 2.4 A — Available（可获取性）

**定义：** 数据记录**必须**在需要时能够及时获取、检索和审查。

**核心要求：**
- 数据**必须**在规定期限内可提供给监管机构审查
- **应当**建立有效的数据检索和备份恢复机制
- 数据归档**必须**便于后续调用

**实施要点：**
- 建立数据目录和索引系统
- 定期测试数据恢复能力
- 确保归档数据与生产环境数据格式兼容

## 3 ALCOA+ 在临床试验中的应用

| ALCOA+ 原则 | 在临床试验中的体现 |
|-------------|-------------------|
| **Attributable** | EDC 电子签名；源文件研究者签名 |
| **Legible** | CRF 清晰可读；电子数据 Human-readable |
| **Contemporaneous** | 实时数据录入；自动时间戳 |
| **Original** | 保留源文件；SDV 核查 |
| **Accurate** | 逻辑核查；双人录入；质疑管理 |
| **Complete** | 100% 数据核查；审计追踪 |
| **Consistent** | 跨中心数据一致性检查 |
| **Enduring** | 长期存档；数据迁移验证 |
| **Available** | 数据锁定与归档；远程数据访问 |

---

## 4 数据生命周期管理

```
数据生成 → 数据录入 → 数据核查 → 数据锁定 → 数据归档 → 数据销毁
   ↓          ↓           ↓           ↓           ↓           ↓
 可归因性   同时性      完整性      持久性      可获取性     -
   -          -       一致性      -           -           -
```

## 5 FDA/WHO 对 ALCOA+ 的立场

- **FDA** 在多个警告信和 483 缺陷报告中引用 ALCOA+ 原则
- **WHO** 在数据完整性指南（TRS 996 附件 5）中明确要求应用 ALCOA 原则
- **MHRA** 在数据完整性行业指南中将 ALCOA+ 作为核心框架
- **NMPA** 在《药品记录与数据管理要求》中体现 ALCOA+ 精神

## 6 常见违规情形

| 违反原则 | 典型情形 |
|----------|----------|
| 不可归因 | 共享账号；匿名记录；无签名的纸质记录 |
| 不清晰 | 字迹潦草；使用代码无说明；划改后不可读 |
| 非同时 | 倒填日期；事后补填；回顾性录入 |
| 非原始 | 直接修改扫描件；删除原始数据 |
| 不准确 | 数据计算错误；选择性录入；伪造数据 |
| 不完整 | 删除失败实验记录；缺失审计追踪 |
| 不一致 | 数据自相矛盾；跨表单数据不符 |
| 不持久 | 使用劣质纸张；格式过时无法读取 |
| 不可获取 | 数据丢失；无法检索；备份失效 |
