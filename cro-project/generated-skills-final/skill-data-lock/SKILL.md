## Constraints / 约束

### Must (必须)
- **C001**: 数据锁定必须满足以下全部条件：
- **C002**: 必须完成全部受试者的计划访视
- **C003**: 必须完成全部安全性和疗效数据的收集
- **C004**: 如果 存在受试者提前退出或失访，应当记录原因并在锁定报告中说明
- **C005**: 必须完成 100% 的源数据核查（SDV）
- **C006**: 必须关闭所有待处理的质疑（开放质疑数量为零）
- **C007**: 必须完成所有方案偏离的评估和关闭
- **C008**: 必须由医学监查员（MA）完成主要疗效数据的医学审核
- **C009**: 必须由数据经理完成数据质量审核报告
- **C010**: 如果 存在尚未解决的医学问题，必须在锁定前完成医学裁决，并形成书面记录
- **C011**: 如果 存在影响主要疗效终点的不良事件，应当由医学监查员确认评估无误后方可锁定
- **C012**: 如果 申办方要求在锁定前进行统计专家审核，应当在满足统计审核要求后执行锁定
- **C013**: 必须由数据经理确认全部检查项目通过后方可发起锁定流程
- **C014**: 应当将检查清单作为锁定报告附件存档
- **C015**: 1. 数据经理确认满足锁定条件后，必须填写《数据锁定申请表》
- **C016**: 2. 锁定申请应当包含以下信息：
- **C017**: 1. 锁定申请必须经以下角色审批：
- **C018**: 2. 如果 任一审批环节不通过，则不得执行锁定操作，需返回修改后重新提交
- **C019**: 1. 审批通过后，必须由授权的数据管理员执行数据库锁定
- **C020**: 2. 锁定操作必须记录执行时间、执行人及锁定确认信息
- **C021**: 3. 锁定完成后，应当向所有相关方发送锁定通知
- **C022**: 解锁必须满足以下条件之一：
- **C023**: 解锁必须经过以下审批程序：
- **C024**: 如果 解锁涉及受试者知情同意相关问题，必须先获得伦理委员会的书面意见
- **C025**: 如果 解锁需要修改统计计划，应当由统计师确认后执行
- **C026**: 如果 解锁涉及跨中心数据，应当评估对整体数据的影响
- **C027**: 解锁后必须重新执行相关核查流程，确认修正正确性
- **C028**: 必须填写《数据解锁记录表》，记录解锁原因、批准人、执行人及解锁范围
- **C029**: 解锁记录应当作为项目档案永久保存

## Decisions / 决策规则

### Decision Table 1 (decision_table_1)

**输入变量**: 序号, 检查项目, 负责角色
**输出变量**: 状态

**规则**:
- 当 序号=1, 检查项目=全部受试者数据录入完成, 负责角色=数据录入员 → 则 状态=☐
- 当 序号=2, 检查项目=全部质疑已关闭, 负责角色=DMC → 则 状态=☐
- 当 序号=3, 检查项目=全部方案偏离已评估, 负责角色=DMC/医学监查员 → 则 状态=☐
- 当 序号=4, 检查项目=医学数据审核通过, 负责角色=医学监查员 → 则 状态=☐
- 当 序号=5, 检查项目=数据质量报告已完成, 负责角色=数据经理 → 则 状态=☐
- 当 序号=6, 检查项目=盲态维护（如适用）已确认, 负责角色=统计师 → 则 状态=☐
- 当 序号=7, 检查项目=锁定批准文件已签署, 负责角色=数据经理/QA → 则 状态=☐
### Decision Table 2 (decision_table_2)

**输入变量**: 审批级别, 适用场景
**输出变量**: 审批人

**规则**:
- 当 审批级别=一级解锁, 适用场景=修正少量非关键数据错误 → 则 审批人=数据经理批准
- 当 审批级别=二级解锁, 适用场景=修正影响分析集划分的数据 → 则 审批人=数据经理 + QA 批准
- 当 审批级别=三级解锁, 适用场景=重大安全性或疗效数据修正 → 则 审批人=数据经理 + QA + 申办方 + 伦理委员会
### If-Then Decision Rules (decision_table_if_then)

**输入变量**: condition
**输出变量**: action

**规则**:
- 当 condition=任一审批环节不通过 → 则 action=不得执行锁定操作，需返回修改后重新提交
