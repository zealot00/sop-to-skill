# SOP/Skill 合规性审计检查清单

## 一、完整性检查

### 1.1 GxP 覆盖检查
- [ ] 是否包含 21 CFR Part 11 的全部要求？
- [ ] 是否覆盖 EU Annex 11 的关键条款？
- [ ] ALCOA+ 原则是否全部体现？

### 1.2 约束词覆盖检查
- [ ] "必须" 约束是否完整？
- [ ] "应当" 约束是否合理？
- [ ] "可以" 约束是否有正例和反例？

### 1.3 角色权限检查
- [ ] 是否定义了所有涉及角色？
- [ ] 角色权限是否清晰？
- [ ] 是否有越权风险？

## 二、准确性检查

### 2.1 语义一致性
- [ ] Skill 约束与 SOP 原文语义是否一致？
- [ ] 条件分支是否正确？

### 2.2 数值准确性
- [ ] 数值是否准确（时间、金额、百分比）？
- [ ] 单位是否正确？

### 2.3 法规引用
- [ ] 法规引用是否准确？
- [ ] 版本是否最新？

## 三、可执行性检查

### 3.1 约束可执行性
- [ ] 约束是否可被系统验证？
- [ ] 是否有歧义？

### 3.2 边界条件
- [ ] 边界值是否清晰？
- [ ] 边界情况是否有处理？

## 四、冲突检查

### 4.1 内部冲突
- [ ] 同一 Skill 内是否有冲突约束？
- [ ] 约束之间是否有矛盾？

### 4.2 外部冲突
- [ ] 与其他 Skill 是否有冲突？
- [ ] 优先级是否正确？

## 五、风险评估

### 5.1 高风险场景
- [ ] 数据完整性风险
- [ ] 合规风险
- [ ] 操作风险

### 5.2 缓解措施
- [ ] 是否有预防措施？
- [ ] 是否有检测机制？
